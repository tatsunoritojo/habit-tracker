## 1. エグゼクティブサマリー

### 1.1 一言で言うと

**「繋がりすぎない繋がり」で習慣形成を支援するアプリ**

### 1.2 解決する課題

- 習慣を身につけたいが、一人では続かない
- 既存SNSは情報量が多く疲れる、でも誰かの存在は感じたい
- 習慣アプリは孤独、SNSはコミュニケーションコストが高い

### 1.3 提供する価値

- **1タップで記録**: 毎日の習慣達成を最小限の操作で記録
- **軽い繋がり**: テキスト/音声なし、リアクションだけで「誰かと頑張っている感」を提供
- **自動マッチング**: 同じ習慣カテゴリの人と、アプリが自動で繋げる

## 2. 背景と市場機会

### 2.1 市場環境

| 市場 | 規模（2024年） | 成長率（CAGR） |
|------|----------------|----------------|
| 習慣トラッカー（グローバル） | 約15〜20億ドル | 13〜15% |
| ウェルネスアプリ（日本） | 約6.9億ドル | 17.3% |
| フィットネスアプリ（日本） | 約25億ドル | 20.5% |

### 2.2 市場機会

- 習慣・ウェルネス・SNS市場はいずれも二桁成長
- 「匿名・低負荷・リアクションのみ」のポジションに明確なリーダー不在
- 「SNS疲れ」と「自己改善ブーム」の交差点にニーズがある

### 2.3 競合状況

| カテゴリ | 代表例 | 本アプリとの違い |
|----------|--------|------------------|
| 習慣トラッカー（ソロ） | Loop, Streaks | ソーシャル機能がない |
| 習慣トラッカー（ソーシャル） | Habitica, Habitify | チャット/コメント前提で重い |
| 軽量SNS | BeReal | 画像投稿必須、習慣特化ではない |
| 勉強垢文化 | X, Discord, LINE OC | テキストコミュニケーションが重い |

**本アプリのポジション**: 「Loopの記録の軽さ」+「Habiticaの他者の存在感」の中間、ただしテキストなし

## 3. ターゲットユーザー

### 3.1 ペルソナA：自己投資志向の20〜30代

- 語学・資格・筋トレ・読書など「習慣にしたいこと」が複数ある
- 既存SNSは情報量が多くて疲れるが、「誰かと頑張っている感じ」は欲しい
- Notionやタスク管理アプリを使い倒そうとして挫折した経験を持つ

### 3.2 ペルソナB：SNS疲れ気味のGen Z

- TikTok・Instagramは日常的に使うが、比較によるメンタル負荷を感じている
- 完全に一人は寂しく、「同じことをしている誰か」の存在は欲しい
- 承認欲求ではなく「ゆるい共感」を求めている

### 3.3 ペルソナC：時間のない30〜40代

- 健康診断の結果から危機感はあるが、長時間のトレーニングや日記は続かない
- 家事・育児・仕事で時間はないが、1〜3分の小さな習慣なら現実的
- 「最低限の自己投資インフラ」を求めている

## 4. プロダクトコンセプト

### 4.1 コアコンセプト

**「繋がりすぎない繋がり」**

- テキスト/音声コミュニケーションを排除
- リアクション（エール）のみで「人の存在を感じられる」程度の繋がり
- アプリ主導のマッチングで、ユーザーが能動的に探さなくてよい

### 4.2 設計原則

1. **軽さ最優先**: 1タップで記録完了、コミュニケーションコストゼロ
2. **匿名性**: プロフィール不要、記録とリアクションだけの関係
3. **アプリ主導**: マッチングもエール提案もアプリがコントロール
4. **ポジティブ体験**: 失敗を責めない、再開を称える

### 4.3 行動心理学に基づく設計

本アプリは行動科学の知見を設計に取り入れています。

| 理論 | アプリへの適用 |
|------|----------------|
| Fogg Behavior Model | 1タップ記録で「実行しやすさ」を最大化 |
| Habit Loop | エールを「報酬」として習慣ループを強化 |
| 自己決定理論 | 外発的報酬より「成長感」「共走感」を優先 |
| セルフ・コンパッション | 失敗を責めず、再開を称える設計 |

## 5. 機能概要

### 5.1 行動継続機能（コア）

| 機能 | 説明 |
|------|------|
| カード作成 | 継続したい習慣を1枚のカードとして登録 |
| 1タップ記録 | カードをタップするだけで当日の達成を記録 |
| 複数カード管理 | 複数の習慣を並行して管理可能 |
| カレンダー振り返り | 達成日に色がつくカレンダー表示 |

### 5.2 カード設計

| 項目 | MVP | 将来 |
|------|-----|------|
| テンプレート | ○ アプリ側が用意 | 拡充 |
| ユーザー作成 | × | ○ 後から追加 |
| カテゴリ構造 | ○ 3階層 | - |
| カード共有 | × | ○ 後から追加 |

### 5.3 SNS機能（軽量ソーシャル）

| 機能 | 説明 |
|------|------|
| 公開/非公開設定 | カード単位で選択 |
| マッチング | 同じカテゴリ（最下層）のユーザー同士、アプリが自動でマッチング |
| エール送信 | ログイン時にアプリから提案 → 1タップでリアクション送信 |
| リアクション | 3種類から選択（テキストなし） |
| 匿名性 | プロフィール非公開、記録とリアクションのみの関係 |

### 5.4 リアクション（エール）設計

UXデザイナー・行動心理学アドバイザーの検討を経て、以下の3種類に確定しました。

| リアクション | アイコン | 意味・使う場面 |
|--------------|----------|----------------|
| ナイス継続 | 💪 | 継続そのものへの励まし。基礎リアクション。 |
| すごい！ | ⭐ | 節目・成長へのお祝い。ハイライト時に。 |
| 一緒にがんばろ | 🤝 | 伴走感・仲間感。同じカテゴリで頑張っている共感。 |

**設計のポイント**
- 3種類に絞り「迷わず選べる」ことを優先
- 競争・比較要素は排除（完璧主義ユーザーへの配慮）
- 「応援」「称賛」「共感」の3軸でポジティブ感情をカバー

### 5.5 カレンダー・統計表示

| 項目 | 設計 | 理由 |
|------|------|------|
| 達成日 | アクセントカラーで表示 | 達成を視覚的に強調 |
| 未達成日 | 無色 or 薄グレー | 「穴」を主張させない、失敗感を軽減 |
| メイン指標 | 「今週○日」「今月○日」 | 累積・増分指標で前向きに |
| ストリーク | カード詳細画面でサブ表示 | 途切れのダメージを軽減 |

### 5.6 再開ケア

| 機能 | 内容 |
|------|------|
| おかえり演出 | 3日以上空いて記録時に「おかえり、○日ぶりに再開！」+ アニメーション |
| 復活バッジ | 中断後に3日再開で「復活の一歩」バッジ獲得 |

### 5.7 エール頻度の調整

| 方式 | 説明 |
|------|------|
| 手動設定 | high / medium / low / off から選択（デフォルト: medium） |
| 将来 | マイクロアンケートで自動調整 |

## 6. ゲーミフィケーション

### 6.1 設計方針

- **軽さ最優先**: Habiticaのような重いRPG化は避ける
- **競争より共走**: ランキング・比較要素は非採用
- **失敗を責めない**: 再開・復活を称える設計

### 6.2 MVP採用要素

| 要素 | 内容 |
|------|------|
| バッジ（3種） | 「はじめの一歩」「1週間チャレンジ」「復活の一歩」 |
| ストリーク表示 | カード詳細でサブ表示（ホームでは主役にしない） |
| エールカウント | カード単位で累計受信数を表示 |

### 6.3 非採用要素

| 要素 | 理由 |
|------|------|
| レベル/経験値 | 軽さを損なう |
| ランキング | 競争・比較を煽り、SNS疲れと相性悪い |
| 連続日数リセット強調 | 完璧主義を助長、離脱リスク |

## 7. オンボーディング

### 7.1 設計方針

- **「まず1つだけ」を強く推奨**: 複数習慣の同時立ち上げは失敗しやすい
- **「小さく始める」を促す**: テンプレートは「1分で終わる」レベルを推奨
- **初週でエール体験を完了**: 送受信両方を体験させる

### 7.2 1週間の体験設計

| Day | 体験 | アプリの働きかけ |
|-----|------|------------------|
| 0 | 初回起動、1カード作成、お試しタップ | 「まず1つだけ」を強調 |
| 1-2 | リマインダー受信、記録継続 | 初回エール受信体験 |
| 3-4 | エール送信体験 | ホームにバナー「仲間にエールを送ってみませんか？」 |
| 5-6 | 途中経過の振り返り | 「この1週間で○日タップしました」表示 |
| 7 | 1週間の節目 | 演出 + 2枚目カード提案（任意） |

## 8. 差別化ポイント

| ポイント | 内容 | 競合との違い |
|----------|------|--------------|
| テキスト/音声排除 | リアクションのみのコミュニケーション | Habitica等はチャット前提 |
| アプリ主導マッチング | ユーザーが探さなくてよい | 競合は手動参加が多い |
| 高い匿名性 | プロフィール不要 | BeReal等は画像/友達が必要 |
| カテゴリベースマッチング | 「人」でなく「行動」起点 | 競合は人間関係起点 |
| 行動心理学に基づく設計 | 失敗を責めない、再開を称える | 競合はストリーク途切れを強調しがち |

## 12. リスクと対策

| リスク | 影響 | 対策 |
|--------|------|------|
| 習慣アプリの「飽き」問題 | 3週間〜3か月で離脱 | 行動心理学に基づくUX設計（再開の前向き強調、穴の見せ方） |
| リアクションの単調さ | エンゲージメント低下 | アニメーション演出、将来的なバリエーション追加 |
| マッチング品質 | 不快感、離脱 | アルゴリズム精緻化、1日上限、ブロック機能 |
| プライバシー懸念 | 信頼失墜 | 匿名設計、明確なポリシー |
| エールが届かない孤独感 | 離脱 | 新規ユーザーへの優先配信、「誰から」を見せない設計 |
